## 1. 实时跟踪日志

**这是我用最多的命令，没有之一。**

```bash
openclaw logs --follow
```

开了之后，AI 每一步在干什么、调用了什么工具、读取了什么文件——全部实时显示在终端。

> 就像给 AI 装了个行车记录仪，它走过的每一步都有迹可循。

**什么时候好用？**

- AI 执行任务突然卡住，不知道它在干嘛
- 想看看它到底有没有正确理解你的需求
- 调试工作流的时候trace执行路径

其他相关命令：

| 命令 | 用途 |
|------|------|
| `openclaw logs` | 看历史日志 |
| `openclaw logs --json` | JSON 格式，信息更详细 |
| `openclaw gateway --verbose` | 启动时显示详细过程 |

## 2. 流式回复

这个设置治好了我的“等待焦虑症”。

```bash
openclaw config set channels.feishu.streaming true
```

**作用**：AI 回复像打字机一样逐字显示，不用等全部生成完才看到内容。

✅ **我的评价**：体验提升明显，而且是质变。

以前：发个长Prompt，等个十几秒，看到一堆正确的废话，气得想砸键盘。

现在：一边生成一边看，发现方向不对立即 `/stop`，省时省气。

---

## 3. 开启耗时显示

```bash
openclaw config set channels.feishu.footer.elapsed true
```

每次回复末尾会显示耗时，比如：

> 已完成 · 耗时 1m 54s

**这有啥用？**

- 心理有底，知道这波烧了多少钱
- 发现某次响应异常慢，可以去查原因
- 配合上下文压缩设置，能精准感知 Token 消耗

💰 看着时间，你就知道钱花在哪了。

---

## 4. 开启状态展示

```bash
openclaw config set channels.feishu.footer.status true
```

显示“已读”“正在思考”“正在执行”等状态。

**说真的，没开这个之前，我经常以为 AI 挂了。**

尤其是跑长任务的时候，看着屏幕不动弹，心里犯嘀咕：到底是卡住了，还是在干活？

开了之后，状态一目了然。交互体验提升一个档次。

---

## 5. 群聊@才回复

如果你把 OpenClaw 拉进了群，这个设置能救命：

```bash
openclaw config set channels.feishu.requireMention true --json
```

**作用**：只有被 @ 时才回复，不会在群里刷屏。

> 血泪教训：我之前没开这个，一个200人的群，每条消息都触发 AI 回复，Token 烧得那叫一个快……

💡 **建议**：群活跃度高的，这个一定要开。

---

## 6. 话题独立上下文

```bash
openclaw config set channels.feishu.threadSession true
```

飞书群聊的话题模式下，每个话题拥有独立上下文，互不干扰。

**适用场景**：团队多任务并行，不同话题讨论不同事情，不会串台。

👥 我们团队现在4个人同时用，各聊各的，互不影响，爽歪歪。

## 4条必会指令

上面是设置，这4条是**聊聊天就能触发**的快捷指令：

| 指令 | 作用 |
|------|------|
| `/stop` | 中止当前任务，立即停止 |
| `/status` | 查看健康状态、上下文用量 |
| `/compact` | 压缩上下文，清理历史 |
| `/new` | 新建会话 |

**我的使用习惯：**

- AI 答非所问 → `/stop` 打断
- 对话太长想省 Token → `/compact`
- 开始新任务 → `/new` 清爽开局

## 进阶：这样配置能省30%-50%的钱

上面那些是“会用”，下面这些是“省着用”：

### 上下文压缩

```bash
openclaw config set agents.defaults.compaction.mode "default"
```

**效果**：旧对话自动压缩成摘要，不原样保留。

📊 实测：省 **30-40%** Token。对话一长，效果立竿见影。

---

### 上下文修剪

```bash
openclaw config set agents.defaults.contextPruning.mode "cache-ttl"
openclaw config set agents.defaults.contextPruning.ttl "4h"
```

**效果**：超过4小时的工具输出自动清理，避免无用信息占地方。

📉 预计省 **20-30%**。
