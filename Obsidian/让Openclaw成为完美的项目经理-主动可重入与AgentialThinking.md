# 让Openclaw成为完美的项目经理-主动可重入与AgentialThinking

TL;DR

• 调教 AI 就像培训员工，需要定义规则（AGENTS.md）
• 核心能力：主动性、可重入性、Agential Thinking
• 与 Claude Code 协作：共享工作记忆日志

---

## 像培训员工一样调教 AI

就像所有刚被招进团队的实习生，OpenClaw 也不是开箱即用的，你需要去培训它：

• 什么事你能做？
• 什么事要让其他 Agents / 人类同事搞定？
• 进度怎么汇报？
• 哪些规则要遵守？

这些"规则"，在 OpenClaw 里是一个叫做 AGENTS.md 的文档，是一个定义了 AI 所有行为准则的配置文件。

基础配置很简单，就像一个没有培训过的新员工。但不断优化配置文件，让它成长成了可以协调其他 Agents 同事的 COO。

---

## 1/ 主动性

很多 Agents 之所以被大家觉得是工具、只是机器人，是因为它们是被动的——你给它一个指令，它执行一次。

它并不会主动想办法解决遇到的问题，也不会主动去思考复盘怎么把一件事做得更好。这样的 Agent 只能当一个螺丝钉。

而 OpenClaw 不同，它本身有一个心跳机制「HEARTBEAT」，会定时检查：

• 哪些项目卡住了？
• 哪里遇到它解决不了的问题？
• 并主动跟人类汇报

在 AGENTS.md 里配置更多规则，让它能做到——不需要人类主动问任何一个项目的进度。

---

## 2/ 可重入性

Agent 稳定、靠谱，并不是说它可以有百分之一百的 uptime，它不需要永远保持在线。

因为项目的推进永远会出现问题，再好的网络也会有意外的中断。

要的是：

当它重启重入的时候，能够从断掉的地方稳定地继续向前推进，甚至可以知道是为什么出现了中断的问题，并去修复这个问题。

OpenClaw 的工作记忆管理是被动进行的——也就是说，如果它认为对话没有涉及重要的内容，它就不会把它写到工作日记当中。

这很容易导致 gateway 重启后，它就忘了之前在干嘛。

优化了它主动压缩上下文、主动刷新 sessions 的配置，让它能够在横跨多天的对话中，依然清楚知道不同项目应如何推进。

---

## 3/ Agential Thinking

调教 agent 来与人协作，应该让 agent 像人类一样去思考？

恰恰相反——硅基生物就应该用它们自己的方式来解决问题。

### 举个例子

让 Agent 去推特上发一条帖子，agent 有很多技能可以调用：

| 方式 | 效率 |
|------|------|
| 像人一样操控浏览器，模拟点击和键盘输入 | ❌ 最慢，最浪费 Token |
| 写一段 JS/Playwright 脚本，调试后运行 | ⚠️ 中等 |
| 直接调用发布推文的 API | ✅ 最高效（省钱+快速） |

最慢最浪费 Token 的方式，就是按人类的方式来。

在它的行为准则当中，明确规定了它不同工具 tool-use 的优先级，不仅是当它自己要去解决问题的时候，也是当它要和其他的 agents 交互信息的时候。

---

## 与 Claude Code 协作

工程师就是 Claude Code。

OpenClaw 去和 Claude Code 共享项目 context 的时候，根本不需要像人类一样开会聊天，它只要把自己在这个项目上的工作记忆日志，开放给 Claude Code 就可以了。

多 Agents 共享工作区记忆、Skills 等工具。

---

## 总结

整个 Agents 团队的协调枢纽和核心，就是 OpenClaw。

调教关键点：
1. 主动性——主动检查项目状态
2. 可重入性——重启后能继续工作
3. Agential Thinking——用最高效的方式解决问题
